30 Seconds

30 秒版本怎么讲

ExecGov 不是普通聊天系统，而是一套让 AI 在规则内调用真实能力和流程的智能执行平台。它把分散在文件夹、服务器和个人经验里的能力，收成一个可控、可交付、可持续更新的统一入口。当前最先落地、最成熟的第一种能力载体，是 Python 脚本。

3 Minutes

3 分钟版本怎么讲

先讲问题：很多企业已经有脚本、接口和流程，但没有统一执行入口。再讲平台：AI 负责理解需求和推荐能力，平台负责边界、确认、执行和审计。最后讲价值：一个客户一个租户，一个客户一个网址，能力可以持续更新，而当前第一阶段主要以脚本型能力落地。现在更重要的是先把真实问题解决好，而不是把路线讲得很远。

Do Say

对外表达要点

  • 它把分散脚本、接口和流程收进统一入口
  • AI 在规则内推荐和触发能力
  • 平台强调租户隔离、确认执行和审计留痕
  • 一个客户一个租户，一个客户一个网址
  • 先讲当前可用能力，再讲后续扩展方向

Current Stage

当前阶段适合这样理解

  • 平台当前最成熟的执行资产是 Python 脚本
  • execgov-cli 适合本地接入和试用，常驻 Agent 能力在持续完善
  • 正式客户交付支持独立租户、独立网址和持续更新
  • public 超管侧已先落下一版 HTTP 能力接入样板，验证第二种能力如何进入治理链路
  • 更标准的小团队使用方式会继续收敛
  • 更多能力类型会逐步接入统一治理链路
