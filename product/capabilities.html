核心能力 ​

1. 多租户隔离 ​

平台当前采用 PostgreSQL schema 级多租户模型：

  • public 负责平台治理和共享底座
  • 每个正式客户租户拥有自己的独立 schema
  • 租户之间的脚本、Skill、日志和系统数据默认隔离

这让平台可以同时承接多个客户，而不是把所有业务混在一个后台里。

2. 智能大脑执行闭环 ​

平台强调的是“可信执行”，不是“随便聊天”。

当前闭环包括：

  • 意图识别
  • 计划生成
  • 人工确认
  • 脚本 / Skill 执行
  • 执行结果返回
  • 执行留痕与审计

3. 脚本注册与 Skill 映射 ​

平台当前已经打通：

  • 脚本注册入库
  • Skill 可视化查看与编辑
  • Skill 动作编排
  • 租户级启停与绑定

这意味着平台不再依赖纯硬编码脚本调用，而是在往正式能力目录演进。

4. 平台模板分发 ​

平台模板已经进入正式链路：

  • 平台模板发布
  • 租户安装申请
  • 租户确认安装
  • 版本管理
  • 回滚
  • 分发日志

这条链路是后续模板市场、团队版和企业版升级的重要基础。

5. 公告与日志治理 ​

平台已经具备：

  • 平台公告与租户公告
  • 指定租户 / 指定用户投放
  • 已读状态与已读明细
  • 登录日志 / 操作日志 / 平台审计

这让产品不只是能执行，还能治理。

6. CLI 与本地接入 ​

smartflow-cli 当前已经完成最小闭环：

  • login
  • register
  • list
  • run
  • agent describe
  • agent start

它当前的意义不是“命令行包装器”，而是平台和本地环境之间的桥梁。
