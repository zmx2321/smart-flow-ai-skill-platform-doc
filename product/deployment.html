Deployment

部署方式

Deployment Thesis控制面可以统一，执行位置不必统一。

这正是 SmartFlow 能同时承接企业交付、产品体验和本地执行的关键原因。

SmartFlow 的部署思路不是只有一种，而是根据 数据边界、客户规模和执行位置 做组合。 平台的目标不是强迫所有用户走同一条路径，而是用同一套治理逻辑覆盖不同落地形态。

01

单租户交付模式

  • 适合企业客户项目交付
  • 需要独立访问地址
  • 需要和其他客户完全隔离
  • 一个客户一个租户，能力与数据边界清晰

02

平台治理模式

  • 平台方统一管理多个客户
  • 需要公告、模板、日志和台账总控
  • 需要跨租户治理但不打破隔离
  • public 作为平台治理域

03

免费入口模式

  • 适合产品体验入口和个人开发者试用
  • 更强调上手速度和体验路径
  • 为后续团队版、企业版升级提供前置入口

04

本地混合执行模式

  • 适合文件和数据不能全部上云
  • 任务依赖本地环境或内网资源
  • 平台负责入口、规划、权限和审计
  • 本地环境负责执行，CLI / Agent 负责桥接

05

未来边缘与设备模式

  • 这是 4.0 的公开方向之一
  • 能力单元不再局限于脚本
  • 执行端可能是本地 Agent、边缘节点或设备插件
  • Web 控制面继续作为统一治理入口

Upgrade Path

从免费入口到正式交付，当前更真实的升级路径

阶段当前状态更适合的部署理解下一步信号
01. 公开体验期先看公开体验页、登录页、注册页和公开文档这时重点不是上来就谈私有化，而是先确认执行链路、产品定位和入口体验是否成立如果要反复使用，就进入免费入口或继续做方案沟通
02. 个人免费入口期用户已经进入免费入口，开始理解技能库、AI 执行和基础使用路径这时更接近产品体验与增长入口，不等于已经进入正式团队 / 企业交付如果开始关心多人协作、客户隔离、正式验收，就进入评估期
03. 方案评估期开始带着真实脚本、真实场景、真实权限问题来沟通这时通常要同时看 客户流程 / 交付、CLI 指引 和部署方式，而不是只看免费线路如果需要独立网址、独立租户和正式验收，就进入单租户交付
04. 正式交付期客户要拿到自己的访问地址、自己的能力边界和正式交付包当前主线是单租户交付：一个客户一个租户，一个客户一个网址如果要统一管理多个客户，再叠加平台治理能力
05. 本地扩展期任务开始依赖本地环境、内网资源或私有数据在单租户交付或平台治理之上叠加本地混合执行，由 CLI / Agent 承接桥接如果还要进一步走标准化梯度，就看 版本路线

Upgrade Signal

哪些信号说明你该从免费入口升级到正式交付

  • 需要一个客户自己的正式访问网址
  • 需要独立租户和更明确的数据 / 能力隔离
  • 需要正式账号、权限、验收和后续更新说明
  • 需要把真实脚本、真实业务场景和真实审批边界接进来
  • 需要把平台从体验入口变成可持续交付空间

Current Truth

当前外部口径里要说清的现实

  • 免费入口当前承担的是体验和增长，不是正式企业交付的最终形态
  • 当前真实收入主线仍然是企业 1.0 单租户交付
  • 标准团队版 / 套餐化梯度属于 2.0 方向，不应讲成已经完全自助开放
  • 所以当前最真实的升级路线是：免费入口感知价值，再进入正式交付沟通

Standard Delivery

标准交付内容

  • 一个独立租户
  • 一个独立访问网址
  • 一组客户专属脚本能力
  • 平台账号与基础权限
  • 客户使用说明与脚本热更新说明

Current Boundary

当前默认不包含什么

  • 面向所有客户开放的公共市场
  • 客户绕过平台自行发布脚本
  • 无审批的高风险自动执行
  • 打破租户边界的跨客户能力共享

How To Choose

如何选择当前最适合你的部署路径

Customer Delivery如果你当前重点是客户交付

优先单租户交付模式，再视情况补 public 治理能力。

Platform Ops如果你当前重点是平台运营

在租户隔离之上叠加平台治理模式，统一公告、模板、日志和台账。

Delivery Ops如果你要看客户到底怎么上线和后续怎么更新

直接看客户流程 / 交付页，把标准交付包、使用路径、热更新和验收重点一次看清。

Growth Entry如果你当前重点是获客与试用

加上免费入口模式，让访客可以先体验，再进入团队版或企业版路径。

CLI Bridge如果你当前重点是本地环境、内网和私有数据执行

先看 CLI 指引，把 login / register / run / agent 的当前接入边界和落地路径跑通。

Long Term如果你还要考虑长期设备化延伸

提前预留边缘与设备模式，为 4.0 的能力包与设备插件治理做准备。
