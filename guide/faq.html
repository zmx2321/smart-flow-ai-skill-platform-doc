Positioning

定位相关

它是不是一个普通聊天系统

不是。它是一个让 AI 在规则内调用和执行真实能力的平台，当前最成熟的执行资产是 Python 脚本。

它适合哪些人

适合已经有脚本、自动化任务、数据处理需求，或需要把这些能力变成统一入口、统一交付和统一治理的人。

免费版是不是企业后台裁剪版

不是。免费版面向个人探索者，用来快速体验能力接入、AI 调度与执行反馈；团队与企业场景再进入更完整的协作和治理路径。

企业线和免费线是不是两套完全分裂的系统

不是。它们共享同一套执行与安全底座，只是面向的使用规模、协作深度和交付方式不同。

Safety

安全、隔离与权限边界

AI 会不会越权执行

当前平台强调 AI 在规则内工作。推荐范围、执行边界、人工确认和审计留痕都不是可有可无的装饰层。

一个客户能不能看到另一个客户的能力

不能。当前正式交付模型是一个客户一个租户，客户默认只能看到自己租户内被授权的能力。

平台如何保证不同客户之间隔离

平台默认按客户边界隔离能力、数据和权限，确保不同客户之间互不可见、互不干扰。

AI 会不会跨租户推荐别的客户能力

不会。当前推荐范围受租户与权限共同约束，不是“模型想到什么就推什么”。

Delivery

交付与热更新

是不是一定要私有化才能用

不一定。可以根据数据边界、客户规模和执行位置，选择单租户交付、平台治理组合或混合执行路径。

热更新是不是要重部署整个平台

不是。客户脚本通过审核、同步、注册和授权后，可以在当前租户内生效，不需要整个平台重新部署。

客户能不能自己上传脚本后直接正式上线

当前 1.0 默认不这样做。平台仍保留审核、注册、授权和权限验证流程，不支持绕过平台直接正式上线。

热更新会不会影响其他客户

不会。脚本更新后只在所属租户范围内生效，其他租户不会自动获得可见或可执行权限。

是不是所有项目都默认包含文件上传和结果下载

不一定。如果项目需要处理输入文件和结果产物，平台可以提供文件上传、批次状态和结果下载链路；是否开放以具体项目范围为准。

正式交付后客户通常会拿到哪些说明

通常至少包括访问入口、账号方式、客户使用说明、客户脚本热更新说明；如果项目包含公告、客户须知或结果下载入口，也应一起交待清楚。

CLI & Local

CLI、本地脚本和 Agent

CLI 现在是不是概念功能

不是。execgov-cli 已支持 login、register、list、run、agent describe 和 agent start，适合本地接入、调试和试用。

本地 Agent 是不是已经正式商用

还不是。本地 Agent 相关能力仍在持续完善，目前可以先按本地桥接与执行骨架理解。

本地脚本和内网资源能不能接入平台

可以，但通常不是让浏览器直接碰本地路径，而是通过 CLI / Agent 路线承接本地执行桥接。

是不是所有脚本类型都已经完整支持远程入库

远程注册目前主要围绕 Python 脚本，更多类型会继续扩展。

Stage

当前阶段与升级路径

现在已经做出什么

企业 1.0 交付主线、个人免费入口、CLI 最小桥梁能力和外部文档都已可用，可以支撑演示、试用和部分真实场景。

标准化小团队方案是不是已经完全开放

标准化小团队方案、套餐化和更完整的产品能力会继续扩展。

体验页适合帮我判断什么

它适合先感受执行链路、理解平台定位，并判断自己更适合个人注册、团队试用还是企业咨询。

注册成功后会去哪里

当前成功后由提示框承接，再跳回登录页，而不是跳到独立欢迎页。
