01

它不是普通聊天系统

AI 在这里主要承担裁决入口和能力推荐角色，真实目标是让脚本与流程进入可控执行链路。

02

它已经有公开入口

你现在可以直接使用体验页、登录页、注册页和文档站，不需要从演示入口跳转。

03

它已经有交付主线

当前已经形成“标准 SaaS 共享租户 + 企业单租户交付”两条现实路径，不再只有一种正式接入方式。

04

它已经有 CLI 桥接

execgov-cli 当前已经能覆盖登录、注册、列清单、执行和 Agent 骨架入口。

Path First

先判断你现在在哪一层

当前状态更适合的路径先看什么不要急着做什么
第一次接触平台先走免费线体验页、注册页、快速开始不要一开始就按企业项目方式理解
已经在个人空间持续接脚本先判断本地脚本位是否要扩容支付与会员、本地脚本位说明不要把轻量增购当成正式团队订阅
准备进入多人协作先走团队版正式租户入口支付与会员、客户流程 / 交付不要默认自己一定要直接私有化
涉及内网、更强隔离或正式项目验收直接进入企业交付沟通部署方式、客户流程 / 交付、联系页不要继续沿用个人入口或标准会员页口径

Current Entry

当前公开入口怎么理解

入口当前状态适合谁说明
/experience当前正式公开体验页第一次体验的人它是真实入口，不是只用于截图演示的占位页。
/login当前正式登录页已有账号的人用于进入个人免费版或后续正式租户空间。
/register当前正式注册页还没有账号的人注册成功后当前会进入独立结果页，再回到登录与个人主链路。
/register/result当前正式结果页刚注册完成的人专门承接注册成功后的下一步，不再只靠成功提示框。
产品文档当前已成体系产品、技术、售前讲平台是什么、怎么交付、怎么部署、怎么演进。
CLI 文档当前已成体系开发者、本地接入场景解释当前已实现和未实现的命令边界。
联系页当前正式入口要继续推进的人用于判断是继续免费线、本地脚本位、团队版还是企业交付，不再是泛咨询入口。

Try First

体验路径

  1. 先走公开体验路径，感受 AI 推荐与执行闭环
  2. 再看产品概览，判断这是不是你想要的平台类型
  3. 如果需要长期使用，再注册并登录
  4. 再判断是否需要进入 CLI 或团队 / 企业交付路径

Local Script

本地脚本接入路径

  1. 先看 execgov-cli 当前已经支持哪些命令
  2. 再准备你的脚本目录与 README
  3. 按当前规则理解 register / list / run / agent 的边界
  4. 如果涉及真实项目，再看混合执行和交付边界

Delivery

团队 / 客户交付路径

  1. 先看客户流程 / 交付，理解实际使用和热更新路径
  2. 再看部署方式，判断是否要共享 SaaS、单租户、平台治理或混合执行
  3. 确认适合你的版本与交付方式
  4. 最后通过联系页判断是标准团队版继续推进，还是直接进入企业交付沟通

Before You Start

开始前先知道这些

  • 公开体验页适合先感受执行链路，不承接真实业务写入。
  • 注册后会进入个人智能执行台，可以开始接入自己的第一个脚本。
  • 如果只是个人继续使用但还没进入多人协作，先看本地脚本位增购，不要急着跳企业路径。
  • 如果只是标准能力和较轻协作，可以优先评估共享 SaaS 共享租户路径。
  • execgov-cli 当前已可用，本地接入能力会继续增强。
  • 如果已经明确是团队正式协作或企业项目，直接走团队版 / 企业沟通通常更高效。

Current Stage

当前阶段覆盖哪些使用方式

  • 企业 1.0 交付主线已经成立。
  • 个人免费入口已经成立。
  • 标准 SaaS 共享租户前端已经建立，并在继续收口。
  • CLI 最小桥梁能力已经成立。
  • 标准团队协作能力和更丰富的套餐能力会继续扩展。
