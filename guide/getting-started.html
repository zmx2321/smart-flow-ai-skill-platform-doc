Quick Start

快速开始

Current RealitySmartFlow 现在不是单一聊天页面，而是一套已经有公开入口、交付主线和 CLI 桥梁能力的执行平台。

先把这件事理解清楚，后面所有文档都会更好读。

如果你只想用最短时间看懂 SmartFlow 当前到底能做什么、怎么开始、哪些已经正式成立， 这页就是给你的。它优先讲 当前真实路径，不讲未来想象图。

01

它不是普通聊天系统

AI 在这里主要承担裁决入口和能力推荐角色，真实目标是让脚本与流程进入可控执行链路。

02

它已经有公开入口

当前公开口径里已经有体验页、登录页、注册页和外部文档站，不只是内部样机。

03

它已经有交付主线

企业方向当前以“一个客户一个租户、一个客户一个网址”的交付模型为主线。

04

它已经有 CLI 桥接

smartflow-cli 当前已经能覆盖登录、注册、列清单、执行和 Agent 骨架入口。

Current Entry

当前公开入口怎么理解

入口当前状态适合谁说明
/experience当前正式公开体验页第一次体验的人它是真实入口，不是只给截图演示的占位页。
/login当前正式登录页已有账号的人用于进入个人免费版或后续正式租户空间。
/register当前正式注册页还没有账号的人注册成功当前由成功提示框承接，然后返回登录页，不是独立成功页。
产品文档当前已成体系产品、技术、售前讲平台是什么、怎么交付、怎么部署、怎么演进。
CLI 文档当前已成体系开发者、本地接入场景解释当前已实现和未实现的命令边界。
联系页当前正式入口要继续推进的人用于试用、演示预约、交付沟通和合作咨询。

Try First

如果你只是想先试一遍

  1. 先走公开体验路径，感受 AI 推荐与执行闭环
  2. 再看产品概览，判断这是不是你想要的平台类型
  3. 如果需要长期使用，再注册并登录
  4. 最后再决定是否继续看 CLI 或团队 / 企业交付路径

Local Script

如果你已经有本地脚本目录

  1. 先看 smartflow-cli 当前已经支持哪些命令
  2. 再准备你的脚本目录与 README
  3. 按当前规则理解 register / list / run / agent 的边界
  4. 如果涉及真实项目，再看混合执行和交付边界

Delivery

如果你要推进客户或团队试用

  1. 先看客户流程 / 交付，理解实际使用和热更新路径
  2. 再看部署方式，判断是否要单租户、平台治理或混合执行
  3. 确认当前团队版 / 企业版的口径和阶段
  4. 最后通过联系页进入试用或演示沟通

Current Notes

几条当前必须对齐的真实口径

  • 体验页访问统计当前按浏览器会话记一次，不按唯一 IP 去重。
  • 后端仍会记录 visitor_ip、user_agent 和 referer 作为公开入口日志。
  • 注册成功当前不是单独成功页，而是成功提示后跳回登录页。
  • smartflow-cli 当前已可用，但本地 Agent 仍属于骨架，不应写成正式商用本地常驻执行。

Current Stage

现在已经成立什么，还没完全产品化什么

  • 企业 1.0 交付主线已经成立。
  • 个人免费入口已经成立。
  • CLI 最小桥梁能力已经成立。
  • 标准团队版和更重的产品化套餐仍属于 2.0 方向，而不是当前完全自助开放能力。
